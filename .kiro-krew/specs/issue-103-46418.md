# Design Specification: Add Short Commit Hash to About Command Display

**Issue**: #103  
**Title**: Add short commit hash to about command display  
**Closes**: #103

## Solution Approach

Minimal modification to the `handleAbout()` function in `internal/tui/commands.go` to display the commit hash alongside existing version information. The infrastructure is already in place - we just need to extract the commit hash from the version info map and format it properly.

## Relevant Files

### Files to Modify
- `internal/tui/commands.go` - Update `handleAbout()` function to include commit hash display

### Files Referenced (no changes needed)
- `internal/version/version.go` - Already has `CommitHash` variable and `Info()` function
- `Taskfile.yml` - Already injects commit hash via ldflags in build task

## Team Orchestration

Single-file change with no dependencies on other components. This is a straightforward UI modification that uses existing infrastructure.

## Step-by-Step Task Breakdown

### Task 1: Modify handleAbout() function
**Acceptance Criteria**: 
- Extract commit_hash from version.Info() map
- Truncate to 7-8 characters for display
- Show "unknown" when commit hash is not available
- Insert between Build Date and Go Version lines

**Implementation**:
1. In `handleAbout()` function, after getting `info := version.Info()`
2. Extract `commitHash := info["commit_hash"]`
3. Create display value:
   - If commitHash == "unknown" or empty, use "unknown"
   - Otherwise, truncate to first 8 characters: `commitHash[:8]`
4. Insert new line in content slice: `fmt.Sprintf("  Commit:     %s", displayHash)`
5. Position after Build Date line, before Go Version line
