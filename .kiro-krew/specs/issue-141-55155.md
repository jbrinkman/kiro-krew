# Design Specification: Remove Linux and Windows builds from release pipeline

**Issue**: #141  
**Title**: Remove Linux and Windows builds from release pipeline  
**Closes**: #141

## Solution Approach

Streamline the release pipeline to focus exclusively on macOS ARM64 (Apple Silicon) builds by removing Linux and Windows build targets from both the GitHub Actions workflow and Task build system. This simplification reduces maintenance overhead and ensures quality focus on the single supported platform.

## Relevant Files

### Files to Modify:
- `.github/workflows/release.yml` - Remove Linux and Windows binary references from upload step
- `Taskfile.yml` - Remove `build:linux` and `build:windows` tasks, update `build:release` task

### Files to Review:
- `README.md` - Check for any multi-platform references (none found that need updates)

## Team Orchestration

This is a straightforward configuration change that requires:
1. Build system modifications (Taskfile.yml)
2. CI/CD pipeline updates (.github/workflows/release.yml)
3. Verification that release workflow produces only macOS ARM64 binary

No coordination with other teams or services required.

## Step-by-Step Task Breakdown

### Task 1: Update Taskfile.yml
**Acceptance Criteria:**
- Remove `build:linux` task entirely
- Remove `build:windows` task entirely  
- Update `build:release` task to only call `build:macos` (remove calls to `build:linux` and `build:windows`)
- Verify task still builds macOS ARM64 binary to correct path

### Task 2: Update GitHub Actions Release Workflow
**Acceptance Criteria:**
- Remove `./dist/release/kiro-krew-linux-amd64` from files list in upload step
- Remove `./dist/release/kiro-krew-windows-amd64.exe` from files list in upload step
- Keep `./dist/release/kiro-krew-macos-arm64` in files list
- Workflow should still execute `task build:release` command

### Task 3: Verification
**Acceptance Criteria:**
- Release workflow only produces and uploads `kiro-krew-macos-arm64` binary
- Local `task build:release` only builds macOS ARM64 binary
- No broken references or tasks remain

## Validation Commands

```bash
# Verify Taskfile.yml tasks
task --list | grep build:    # Should show build:release and build:macos only
task build:release           # Should only build macOS ARM64 binary
ls -la dist/release/         # Should show only kiro-krew-macos-arm64

# Verify removed tasks no longer exist
task build:linux 2>&1 | grep "task does not exist"    # Should indicate task doesn't exist
task build:windows 2>&1 | grep "task does not exist"  # Should indicate task doesn't exist

# Verify GitHub Actions syntax
gh workflow view release.yml --yaml | grep -A 10 "files:"  # Should show only macOS binary
```

## Implementation Notes

- Keep the overall structure intact to allow easy re-addition of platforms in the future
- Maintain the `build:release` task as the entry point for consistency
- The `build:macos` task remains unchanged as it already targets the correct platform (darwin/arm64)
- No changes needed to version management or other release workflow steps

## Future Considerations

When re-adding platform support:
1. Add new `build:linux` and `build:windows` tasks to Taskfile.yml
2. Update `build:release` to include new platform tasks  
3. Add corresponding binary paths to GitHub Actions upload step
4. Consider adding platform-specific testing before enabling releases
