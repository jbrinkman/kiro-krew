# Enable Mouse Click Functionality for Agent Tab Close Buttons

**Issue**: #147  
**Root Cause**: TUI program initialized without mouse support enabled  
**Closes #147**

## Problem Statement

The application has complete mouse click handling infrastructure in place, including `HandleTabHeaderClick()` method and tab close logic, but mouse events are not being captured because `tea.NewProgram()` is initialized without mouse support enabled.

## Current State Analysis

### Existing Infrastructure ✅
- **Mouse Click Handler**: `HandleTabHeaderClick()` method exists in `tab_manager.go` with full click position detection
- **Tab Close Logic**: `CloseTab()` methods fully implemented and functional  
- **Event Routing**: Mouse click events properly routed in `Update()` method (lines 325-339)
- **Close Button Rendering**: Tab headers render with "×" close buttons for closable tabs

### Missing Component ❌
- **Mouse Support Initialization**: `tea.NewProgram(mdl)` in `internal/tui/tui.go:923` lacks mouse support option

## Solution Approach

**Minimal Change Strategy**: Add a single program option to enable mouse support during TUI initialization.

The solution requires adding `tea.WithMouseCellMotion()` option to the existing `tea.NewProgram()` call. This enables mouse click event capture while preserving all existing functionality.

## Implementation Details

### File Modifications

**File**: `internal/tui/tui.go`  
**Location**: Line ~923  
**Change Type**: Single line modification  

**Current Code**:
```go
p := tea.NewProgram(mdl)
```

**Updated Code**:
```go
p := tea.NewProgram(mdl, tea.WithMouseCellMotion())
```

### Why `tea.WithMouseCellMotion()`

- Enables mouse click detection required for tab interaction
- Provides cell-level precision needed for close button targeting  
- Compatible with existing mouse event handling code
- Minimal performance impact
- Standard Bubble Tea mouse support option
