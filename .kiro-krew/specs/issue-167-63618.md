# Design Specification: Enhance TUI menu to show contextual stop commands with arguments

**Closes #167**

## Problem Analysis

The current TUI menu system registers the `stop` command without arguments in the command registry, causing a console error when users select it from the autocomplete menu because the stop command requires an issue number argument. The `GetFlattenedMatches` method in the command registry returns the bare `stop` command, which fails when executed.

## Technical Analysis

### Current Architecture

1. **Command Registry** (`internal/tui/command_registry.go`):
   - Maintains a static list of commands and compound commands in `flattenedCommands`
   - The `stop` command is registered with `HasArgs: true` and `ArgPattern: "<issue>"`
   - `GetFlattenedMatches` returns static commands that don't consider runtime state

2. **Autocomplete System** (`internal/tui/autocomplete.go`):
   - Uses the command registry's flattened matches for suggestions
   - Menu selection immediately applies the suggestion and executes on Enter
   - No context awareness of currently running agents

3. **Agent Management** (`internal/agent/manager.go`):
   - `Manager.List()` returns all agents with their status and issue numbers
   - Running agents have `Status == StatusRunning`

## Solution Approach

Modify the command registry to dynamically generate contextual stop commands based on the current state of running agents, implementing the ≤10 vs >10 threshold logic.

### Architecture Changes

1. **Enhanced Command Registry**:
   - Add runtime state awareness to generate dynamic stop commands
   - Implement contextual command generation based on agent count
   - Maintain existing static command structure for other commands

2. **Agent State Integration**:
   - Pass agent manager reference to command registry
   - Query running agents on-demand during autocomplete matching

3. **Template Command Support**:
   - Add support for template commands that position cursor for input
   - Distinguish between executable commands and input templates

## Implementation Notes

### Dynamic Command Generation Algorithm

```
if input starts with "stop":
  runningAgents = getRunningAgents()
  if len(runningAgents) == 0:
    return [] // No stop commands available
  elif len(runningAgents) <= 10:
    return ["stop 123", "stop 456", ...] // Specific commands, sorted by issue
  else:
    return ["stop <issue number>"] // Template command
```

### Template Command Handling

- Template commands are identified by placeholder syntax: `<...>`
- On selection, extract the prefix before the placeholder
- Set input value to prefix and position cursor at end
- Prevent automatic execution

### Performance Considerations

- Agent list queries are lightweight (in-memory operations)
- Limit dynamic command generation to stop commands only
- Maintain existing performance for other command types
