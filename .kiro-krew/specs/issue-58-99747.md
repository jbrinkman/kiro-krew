# Design Specification: Tabbed Agent Output Views

**Issue:** #58 - Implement tabbed views for individual agent output instead of mixed single pane  
**Closes:** #58

## Solution Approach

Enhance the existing tab system to create dedicated tabs for each active agent while maintaining the permanent main TUI tab. The solution leverages the existing `TabManager`, `Tab` interface, and `OutputView` components but extends them to support multiple agent-specific tabs.

### Key Architectural Decisions

1. **Extend Existing Tab System**: Build upon the current `TabManager` and `Tab` interface rather than creating a parallel system
2. **Agent Tab Lifecycle**: Create agent tabs when agents start, allow closing without stopping agent execution
3. **Preserved Agent Output**: Agent output continues to be captured even when tab is closed
4. **Seamless Tab Navigation**: Support both keyboard shortcuts and future mouse interaction
5. **Backward Compatibility**: Maintain F2/O toggle functionality for existing workflows

## Technical Implementation Details

### Agent Tab Structure
```go
type AgentTab struct {
    id          string        // Agent ID
    agentID     string        // Reference to agent
    outputView  *OutputView   // Dedicated output view for this agent  
    manager     *agent.Manager // Reference to agent manager
    styles      *Styles       // UI styling
    width       int
    height      int
}
```

### Tab Navigation Key Bindings
- `F2`/`O`: Toggle between main tab and first agent tab (backward compatibility)
- `Ctrl+Tab`: Next tab
- `Ctrl+Shift+Tab`: Previous tab  
- `Ctrl+W`: Close current tab (if closable)

### Tab Title Format
- Main Tab: "Main TUI"
- Agent Tabs: "Agent #<issue-number>" or "Agent <agent-id>" if no issue
