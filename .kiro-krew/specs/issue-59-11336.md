# Issue #59: Add ability to reopen agent views from status dialog

**Closes #59**

## Solution Approach

Enhance the status dialog to provide clickable access to running agent views. The implementation builds on the existing tabbed interface system from issue #58 and leverages the current agent output capture mechanism to restore views from their current log state.

### Key Architectural Decisions

1. **Extend Status Dialog Functionality**: Enhance the existing `handleStatus()` command to display numbered running agents with keyboard selection
2. **Leverage Existing Tab Restoration**: Use the `TabManager.RestoreOrFocusAgentTab()` method which already handles creating/focusing agent tabs
3. **Preserve Current Log State**: Agent views restore from current captured output buffer, not from the beginning of logs
4. **Maintain Backward Compatibility**: All existing navigation and tab management remains unchanged

### Technical Strategy

The solution integrates with existing systems:
- **Status Dialog**: Already displays running agents, extend with numbered selection
- **Tab Manager**: Already has restoration logic via `RestoreOrFocusAgentTab()`
- **Output Capture**: Already maintains current agent output state in memory
- **Agent Lifecycle**: Already tracks running agents and their states
