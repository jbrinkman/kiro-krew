# Design Specification: Visual Tab Headers for TUI Interface

## Issue: #63 - Add visual tab headers to TUI interface

## Overview
Add visual tab headers to the existing TUI tab system while preserving all current functionality and keyboard navigation.

## Current State Analysis
- TabManager exists with full tab switching logic
- Tab interface defines ID(), Title(), Type(), IsClosable() methods
- Keyboard navigation: `[`/`]` for prev/next, `F2` toggle, `Ctrl+W` close
- No visual representation of tabs currently exists

## Design Requirements

### 1. Tab Header Component
Create a new tab header rendering system that:
- Displays at the top of the TUI interface
- Shows all available tabs with titles
- Highlights the currently active tab
- Shows close buttons (×) for closable tabs (agent tabs)
- Handles terminal width constraints gracefully

### 2. Integration Points

#### A. TabManager Enhancement
- Add `RenderTabHeaders()` method to render visual headers
- Add `HandleTabHeaderClick(x int)` for mouse click handling
- Integrate with existing `SetActiveTab()` method

#### B. Main TUI Integration  
- Modify main TUI view to include tab headers at top
- Adjust content area height to accommodate headers
- Route mouse click events to TabManager

#### C. Styling Integration
- Use existing lipgloss styles for consistency
- Define tab header styles (active, inactive, closable)
- Handle narrow terminal gracefully (truncate/scroll)

### 3. Visual Design

#### Tab Header Layout
```
┌─Main─┬─Agent: agent-1 ×─┬─Agent: agent-2 ×─┐
│                                          │
│            Tab Content Area              │
│                                          │
└──────────────────────────────────────────┘
```

#### States
- **Active Tab**: Bold/highlighted background
- **Inactive Tab**: Normal styling with separator
- **Closable Tab**: Shows × button that responds to clicks
- **Narrow Terminal**: Truncate long titles, maintain functionality

### 4. Implementation Steps

1. **Create tab header renderer**
   - New function in `tab_manager.go`: `RenderTabHeaders() string`
   - Handle styling, truncation, and close buttons

2. **Add mouse click handling**
   - Parse mouse click coordinates
   - Map clicks to tab indices or close buttons
   - Integrate with existing tab switching logic

3. **Update main TUI view**
   - Include tab headers in main view rendering
   - Adjust content area dimensions
   - Route mouse events to TabManager

4. **Preserve existing functionality**
   - All keyboard shortcuts must continue working
   - No changes to Tab interface or existing methods
   - Maintain backward compatibility

### 5. Technical Constraints
- Must work with existing TabManager and Tab interface
- Zero breaking changes to current keyboard navigation
- Follow bubbletea mouse event patterns
- Use lipgloss for styling consistency
- Handle edge cases: no tabs, single tab, many tabs

### 6. Testing Requirements
- Verify all keyboard shortcuts still work
- Test mouse click functionality
- Test narrow terminal handling
- Test with various tab configurations
- Ensure closable/non-closable behavior

## Files to Modify
- `internal/tui/tab_manager.go` - Add tab header rendering and click handling
- `internal/tui/tui.go` - Integrate headers into main view
- `internal/tui/styles.go` - Add tab header styling

## Reference Implementation
Follow patterns from: https://github.com/charmbracelet/bubbletea/blob/main/examples/tabs/main.go
