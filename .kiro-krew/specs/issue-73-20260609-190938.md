# Issue #73: Implement Missing Deterministic Checkers

## Problem Analysis

The `scoreDeterministic` function in `internal/eval/runner.go` currently implements only 3 checkers:
- `completeness` - checks for structural elements (## and ### headers)
- `file_reference` - verifies referenced files exist on disk
- `file_naming` - checks for correct app_docs/feature-* paths

However, 4 deterministic checkers are missing from rubric definitions:

1. `acceptance_criteria_testability` (planner.yaml) - marked `deterministic: true`
2. `test_execution` (validator.yaml) - marked `deterministic: true`  
3. `code_correctness` (builder.yaml) - marked `deterministic: true`
4. `test_coverage` (builder.yaml) - marked `deterministic: true`

## Implementation Strategy

Add cases to the switch statement in `scoreDeterministic` function following existing patterns.

### 1. acceptance_criteria_testability
**Purpose**: Verify acceptance criteria include testable assertions or commands
**Logic**: 
- Look for testable patterns: executable commands, specific file checks, measurable outcomes
- Award higher scores for concrete/executable criteria vs vague descriptions

### 2. test_execution  
**Purpose**: Verify validation commands were executed and results reported
**Logic**:
- Look for command execution evidence: exit codes, command outputs, file system checks
- Check for validation-specific patterns indicating actual test runs

### 3. code_correctness
**Purpose**: Verify generated code compiles/runs without errors
**Logic**:
- Look for build success indicators, absence of compilation errors
- Check for syntax validity patterns, successful test runs

### 4. test_coverage
**Purpose**: Verify appropriate tests are included for new functionality  
**Logic**:
- Look for test file references, test function patterns
- Check for coverage-related output or test execution results
