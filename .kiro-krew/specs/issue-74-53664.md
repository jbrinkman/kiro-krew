# Design Specification: Binary File Protection for Krew-Lead Agent

**Issue**: #74 - Add binary file protection to krew-lead agent commit process  
**Status**: Ready for Implementation  
**Closes**: #74

## Problem Analysis

The krew-lead agent is accidentally committing binary files (executables and build artifacts) to the repository during step 7 (Push Branch) of the workflow. This pollutes the git history with large binary files that should not be version controlled.

**Current Issues:**
- Binary files like `kiro-krew-validate`, `kiro-krew`, `kiro-krew-test` are being committed
- No validation exists before `git add -A` in step 7
- Missing patterns in `.gitignore` (e.g., `kiro-krew-validate`)
- No automated cleanup of accidentally staged binary files

## Solution Approach

Implement binary file detection and protection in step 7 of the krew-lead agent workflow by:

1. **Pre-commit validation**: Check for binary files before committing using `git diff --cached --name-only --diff-filter=A`
2. **Automated cleanup**: Remove detected binary files from both staging area and worktree
3. **Pattern-based detection**: Use file extension and executable detection
4. **Fail-safe approach**: Halt commit process if binary cleanup fails

## Technical Approach

### Binary File Detection Strategy
- Use `git diff --cached --name-only --diff-filter=A` to find newly added files
- Check each file for binary characteristics:
  - Executable permissions
  - Known binary extensions (`.exe`, `.so`, `.dylib`, `.dll`, `.o`, `.a`)
  - Files without extensions that are executable
  - Build artifact patterns (`kiro-krew*`, `*-test`, `*-validate`)

### Safety Mechanism
- Remove from staging: `git reset HEAD <file>`
- Remove from worktree: `rm <file>` (with error handling)
- Log all actions for debugging
- Fail commit if cleanup operations fail

## Implementation Notes

### Binary Detection Logic
```bash
# Check if file is binary
if [[ -x "$file" ]] || [[ "$file" =~ \.(exe|so|dylib|dll|o|a)$ ]] || [[ "$file" =~ ^(kiro-krew|.*-test|.*-validate)$ ]]; then
    echo "Binary file detected: $file"
    # Cleanup logic here
fi
```

### Error Handling Requirements
- Log when binary files are detected and removed
- Fail gracefully if `git reset` or `rm` operations fail
- Continue with commit if no binary files detected
- Provide clear error messages for debugging

### Performance Considerations
- Detection runs only on newly added files (`--diff-filter=A`)
- Minimal overhead for normal commits
- Early exit if no files are staged
